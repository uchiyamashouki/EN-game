# EN-game（Word RPG）

英単語学習をRPG風バトルで進める、フレームワーク非依存のシンプルなブラウザゲームです。  
`index.html` を起点に `main.js` がシーンを切り替え、戦闘・ショップ・学習確認（NPC）を行います。

## 遊び方（操作方法）

### 1. 起動方法
このプロジェクトはビルド不要です。以下のどちらかで起動できます。

- **推奨: ローカルサーバーで起動**
  ```bash
  cd /workspace/EN-game
  python3 -m http.server 8000
  ```
  ブラウザで `http://localhost:8000` を開く。

- **簡易: `index.html` を直接ブラウザで開く**
  - 環境によっては ES Modules の制約で動かない場合があるため、上のローカルサーバー方式を推奨。

### 2. 基本操作フロー
1. 初期画面はバトル画面です。
2. `攻撃 / 回復 / 強攻撃` のいずれかを押すと、英単語クイズ（10問）が始まります。
3. 各問題は **9秒制限**。日本語訳を入力して `決定` か Enter で回答します。
4. 10問の正解数に応じて、技の効果量（ダメージ/回復）が決まります。
5. 敵撃破で報酬（食費）とドロップの可能性があります。敗北時は所持金が半減します。

### 3. 画面内ボタンの意味
- **攻撃**: 標準ダメージ。ステージ範囲の単語から出題。
- **回復**: HP回復。得意単語を優先して出題。
- **強攻撃**: 高威力候補。苦手/未回答単語を優先して出題。
- **中ボス挑戦**: そのステージの習得進捗が一定以上（80%以上）で有効化。
- **NPCに相談**: 得意・苦手単語や正答率の確認画面へ移動。
- 一覧では英単語と日本語訳をセットで確認可能。
- **ショップ**: 回復/防御アイテムを購入。

### 4. クイズ中の挙動
- 正解時: 正解表示と成功エフェクト。
- 誤答時: 時間切れと同様の失敗エフェクト扱いで、正答表示。
- 時間切れ時: 失敗エフェクトと正答表示。

### 5. セーブについて
- プレイデータは `localStorage` に自動保存されます。
- 保存タイミング: シーン遷移時 / 15秒ごとの自動保存 / ページ離脱時。
- 同じブラウザ・同じURLで再度開くと、前回データを読み込みます。

## ファイル構成と役割

### ルート
- `index.html`
  - アプリのエントリHTML。`#app` と `main.js` を読み込み。

### `scenes/`（画面ロジック）

  - ショップ画面。所持金チェック、アイテム購入、在庫（state.inventory）反映。

  - 学習アドバイザー画面。得意/苦手単語と正答率一覧を表示。

## ゲーム設計メモ（ざっくり）
- **1ターン = 10問クイズ**。正解率で行動効果が線形に変化。
- **学習支援**: 正答率に基づき、回復は得意語、強攻撃/ボスは苦手語を優先。
- **進行条件**: ステージ内カバレッジ（正解済み単語割合）で中ボス挑戦可否を制御。
- **成長要素**: 報酬通貨 + ドロップ/購入アイテム。
